# WORKFLOW

Daily working rhythm.

---

## Development loop
1. Clarify intent
2. Decide architecture
3. Implement core logic
4. Delegate mechanical work
5. Review
6. Commit

---

## Order of work
Structure → Content → Styling → Interaction → Polish

---

## Commits
Small, intentional, one idea per commit.

---

## Review checklist
- Mobile
- Dark mode
- Contrast
- Types
- Complexity

---

## When to stop
Stop when work becomes cosmetic or mentally noisy.

---

## Deploy
Deploy only coherent slices.
